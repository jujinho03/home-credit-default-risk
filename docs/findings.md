# Home Credit 분석 보고서

## 1. 분석 목적

대출 신청자의 정보를 이용해 채무불이행 위험을 예측하고, 위험도가 높은 신청자를 우선적으로 구분하는 것이 목표입니다.

처음 수행한 팀 과제에서는 모델 성능과 변수 중요도를 중심으로 결과를 정리했습니다. 제출용 포트폴리오로 다시 구성하면서 다음 질문을 추가로 확인했습니다.

1. 원본 데이터에 오류나 이상한 값이 있는가?
2. 검증 데이터의 정보가 학습 과정에 미리 사용되지 않았는가?
3. 부도 고객이 적은 상황에서 모델을 적절히 평가했는가?
4. 변수 중요도를 잘못 해석한 부분은 없는가?
5. 결과를 금융 업무에서 어떻게 제한적으로 활용할 수 있는가?

## 2. 데이터 확인

분석에는 `application_train.csv` 307,511건을 사용했습니다. 부도 고객은 24,825건으로 전체의 8.07%입니다.

### 주요 점검 결과

- TARGET 결측: 0건
- 신청 ID 중복: 0건
- train/test 신청 ID 중첩: 0건
- 완전 중복 행: 0건
- 수치형 무한값: 0개

`DAYS_EMPLOYED`에서 365243이라는 값이 55,374건 발견됐습니다. 실제 근속일수로 해석할 수 없는 코드이므로 결측값으로 바꾸고, 해당 코드가 존재했다는 사실은 별도 변수로 보존했습니다.

## 3. 학습 데이터 구성

원본 122개 컬럼에서 신청 ID와 TARGET을 제외해 120개 원본 입력 변수를 사용했습니다. 재무 비율, 근속 비율, 외부 신용점수의 평균과 결측 여부 등 13개 파생변수를 더해 전처리 전 133개 입력 변수를 구성했고, 범주형 원-핫 인코딩 후 최종 263차원이 됐습니다.

전체 데이터를 학습 80%, 검증 20%로 나누면서 양쪽의 부도율을 8.07%로 유지했습니다.

| 구분 | 표본 수 | 부도율 |
|---|---:|---:|
| 학습 | 246,008 | 8.07% |
| 검증 | 61,503 | 8.07% |

결측치 대체와 수치형 변수의 표준화 기준은 학습 데이터에서만 계산했습니다. 이는 검증 데이터의 정보가 학습 과정에 미리 들어가는 것을 막기 위한 조치입니다.

부도 고객 수가 적기 때문에 정확도만 사용하지 않았습니다. 데이터는 복제하거나 삭제하지 않고, 모델이 부도 고객의 오류를 더 중요하게 보도록 클래스 가중치를 적용했습니다.

## 4. 모델 비교

학부 통계적 기계학습 수업에서 다룬 5개 분류모델을 비교했습니다.

| 모델 | ROC-AUC | PR-AUC | 해석 |
|---|---:|---:|---|
| Logistic Regression | 0.7504 | 0.2325 | 가장 높은 ROC-AUC |
| L2 Logistic | 0.7504 | 0.2324 | 최종 기준모델 |
| LinearSVC | 0.7502 | 0.2338 | 성능은 비슷하지만 오래 걸림 |
| LDA | 0.7495 | 0.2327 | 선형 모델과 비슷한 성능 |
| RBF SVM | 0.7315 | 0.2067 | 5만 건만 사용한 보조 결과 |

성능 차이가 매우 작았기 때문에 복잡한 모델이 항상 더 좋은 것은 아니라고 판단했습니다. L2 Logistic은 빠르고 비교적 설명하기 쉬워 기준모델로 선택했습니다.

RBF SVM은 계산시간 문제로 전체 학습 데이터가 아닌 5만 건을 사용했습니다. 따라서 다른 모델보다 성능이 낮다고 단정하기보다는 비교 조건이 다르다는 한계를 함께 제시했습니다.

## 5. 변수 중요도 해석 오류 수정

### 발견한 문제

기존 분석에서는 로지스틱 계수의 절댓값이 클수록 중요한 변수라고 정리했습니다. 하지만 수치형 변수는 표준화했고 범주형 더미는 0과 1로 남아 있어 두 종류의 계수를 그대로 비교하기 어려웠습니다.

### 수정 방법

학습 데이터에서 각 변수가 실제로 얼마나 변하는지를 반영해 계수를 다시 비교했습니다. 또한 검증 데이터에서 변수 하나를 무작위로 섞었을 때 모델 성능이 얼마나 떨어지는지도 보조적으로 확인했습니다.

### 수정 결과

- `ORGANIZATION_TYPE_Realtor`: 기존 3위 → 수정 후 93위
- `ORGANIZATION_TYPE_Transport: type 3`: 기존 6위 → 수정 후 88위

드물게 등장하는 조직유형의 중요도가 과장돼 있었다는 것을 확인했습니다. 이에 따라 기존의 조직유형 중심 설명은 사용하지 않기로 했습니다.

대출금액과 상품금액 관련 변수가 상위에 나타났지만 두 변수는 서로 매우 비슷하게 움직입니다. 따라서 한 변수의 계수 부호만 보고 위험의 원인이라고 설명하지 않고, 대출 규모와 관련된 묶음 신호로 제한해 해석했습니다.

## 6. 위험구간 분석

L2 Logistic이 계산한 위험점수를 10개 구간으로 나눴습니다. 가장 위험한 상위 10% 집단의 실제 부도율은 26.48%로 전체 부도율 8.07%의 3.28배였습니다.

이 결과는 한정된 심사 인력을 위험 상위 구간의 추가 서류 확인과 상담에 먼저 배치할 수 있다는 의미입니다. 모델이 개별 고객의 부도를 확정한다는 뜻은 아닙니다.

### 여신 심사 활용과 임계값

부도 고객을 놓치는 대손 위험과 정상 고객을 고위험으로 분류하는 기회손실은 크기가 다릅니다. 실제 산출된 세 임계값을 비교하면 다음과 같습니다.

| 임계값 시나리오 | 임계값 | Recall | Precision | 해석 |
|---|---:|---:|---:|---|
| 기본 임계 | 0.50 | 68.12% | 16.09% | 기준 비교점 |
| F1 최대화 | 0.66 | 42.68% | 23.20% | 정밀도는 높지만 부도 고객 누락이 증가 |
| Recall 80% 이상 | 0.41 | 80.00% | 13.48% | 부도 누락은 감소하지만 추가 심사 대상이 증가 |

대손 위험을 더 크게 본다면 Recall 80% 시나리오를 검토할 수 있지만 실제 비용 금액과 심사 가능 물량이 없으므로 최적 임계값이라고 단정하지 않았습니다. 이 모델은 자동 승인·거절기가 아니라 인적 심사 우선순위 보조도구이며, 신청 테이블만 사용한 학부 프로젝트라는 한계가 있습니다.

## 7. 배운 점

1. 모델 성능을 높이는 것만큼 입력 데이터와 검증 절차를 확인하는 일이 중요했습니다.
2. 같은 숫자라도 변수의 단위와 생성 방식이 다르면 직접 비교하면 안 된다는 점을 배웠습니다.
3. 분석 오류를 감추기보다 원인을 설명하고 수정 과정을 기록하는 편이 결과의 신뢰도를 높였습니다.
4. 분석 결과는 모델 지표뿐 아니라 현업에서 어떻게 사용할 수 있는지와 한계를 함께 설명해야 합니다.

## 8. 한계와 다음 단계

- 이번 모델은 신청 테이블만 사용했습니다.
- 실제 미래 시점 데이터를 이용한 검증은 수행하지 못했습니다.
- RBF SVM은 다른 모델과 학습 조건이 다릅니다.
- 모델 점수는 정확한 부도확률로 해석할 수 없습니다.

다음 단계에서는 과거 신용정보 테이블의 시점을 확인한 뒤 안전하게 결합하고, 시점이 다른 검증 데이터를 사용해 성능 변화를 확인할 수 있습니다.
