# 처리 데이터 안내

`processed/`에는 노트북 실행 중 다시 만들 수 있는 전처리 결과만 저장합니다.

현재 공개 저장소에는 처리 데이터 파일을 포함하지 않습니다. 원본 CSV를 준비한 뒤 노트북을 01번부터 순서대로 실행하면 동일한 전처리 절차를 확인할 수 있습니다.

전처리 원칙은 다음과 같습니다.

- 학습/검증 분할 후 학습 데이터에서만 결측치 대체 기준 계산
- 수치형 변수는 중앙값 대체와 표준화 적용
- 범주형 변수는 최빈값 대체와 원-핫 인코딩 적용
- `DAYS_EMPLOYED=365243`은 결측 처리하고 이상코드 여부를 별도 보존
